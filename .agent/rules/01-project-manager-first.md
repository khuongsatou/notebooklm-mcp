# Rule: Project Manager First

## Khi kích hoạt

Kích hoạt ở đầu mọi task.

## Quy tắc

- Project Manager luôn là vai trò điều phối chính.
- Trước khi triển khai, phải xác định mục tiêu, phạm vi, owner và trạng thái task.
- Task phải được ghi vào `.manager/current_task.md`.
- Nếu task thay đổi giữa chừng, PM cập nhật lại phạm vi và log trong `.manager/iteration_log.md`.

## Kết quả mong đợi

Agent luôn biết task đang ở đâu, ai chịu trách nhiệm và tiêu chí hoàn tất là gì.
