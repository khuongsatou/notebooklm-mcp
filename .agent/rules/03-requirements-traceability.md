# Rule: Requirements Traceability

## Khi kích hoạt

Kích hoạt cho mọi task có yêu cầu chức năng, UI, API hoặc workflow.

## Quy tắc

- Mỗi yêu cầu phải có mã `REQ-001`, `REQ-002`.
- Mỗi thay đổi implementation cần map về ít nhất một requirement.
- QA test case phải tham chiếu requirement tương ứng.
- Nếu có yêu cầu mơ hồ, ghi rõ assumption trong `.manager/requirements.md`.

## Kết quả mong đợi

Không mất dấu lý do tồn tại của từng thay đổi.
