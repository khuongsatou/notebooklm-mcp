# Workflow: QA Validation

## Mục tiêu

Xác nhận task đáp ứng acceptance criteria.

## Các bước

1. QA tạo checklist từ `.manager/requirements.md`.
2. Chạy test hoặc kiểm tra thủ công phù hợp.
3. Ghi pass/fail, evidence và defect vào `.manager/test-report.md`.
4. Nếu fail, tạo feedback hoặc trả về Developer.
5. Nếu pass, chuyển sang UX/Customer Review nếu cần.

## Done khi

Test report có kết luận rõ ràng.
