# Workflow: Requirements Analysis

## Mục tiêu

Biến yêu cầu thô thành requirements có thể triển khai và kiểm thử.

## Các bước

1. PO đọc task hiện tại.
2. Tách functional requirements và non-functional requirements.
3. Gắn mã `REQ-001`, `REQ-002`.
4. Viết acceptance criteria.
5. Lưu vào `.manager/requirements.md`.

## Done khi

Developer và QA có thể hiểu cùng một scope.
